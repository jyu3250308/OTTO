# 🎉 ReddAI Radar: 밈탐정 셜록봇 🎉\n\n![ReddAI Radar Logo](https://img.shields.io/badge/Project-ReddAI%20Radar-blue?style=for-the-badge&logo=python&logoColor=white)\n![Python Version](https://img.shields.io/badge/Python-3.x-green?style=for-the-badge&logo=python)\n![License](https://img.shields.io/badge/License-MIT-purple?style=for-the-badge)\n\n## 🚀 프로젝트 소개: 지루함 타파! 밈 핵폭탄 투하!\n\n**"ReddAI Radar: 밈탐정 셜록봇"**은 지루한 사무실 분위기를 한 방에 날려버릴, 당신의 회사 슬랙 채널을 위한 AI 기반 밈 탐정 봇입니다! Reddit의 실시간 인기 밈과 유머 게시물을 싹싹 긁어모아, 가장 핫한 밈들을 핵심만 요약하여 정기적으로 슬랙에 자동으로 투하합니다. 🕵️‍♂️🔥\n\n더 이상 밈을 찾아 헤매지 마세요! 셜록봇이 알아서 찾아서 배달해 드립니다. 당신의 사무실을 밈으로 물들이고, 월요병을 날려버릴 인싸템을 경험해 보세요!\n\n### ✨ 주요 기능\n\n*   **Reddit 인기 밈 실시간 스캔**: `memes`, `dankmemes`, `funny`, `wholesomememes` 등 지정된 서브레딧에서 인기 게시물을 탐색합니다.\n*   **스마트 필터링**: 특정 키워드(`meme`, `funny`, `joke` 등) 및 플레어(`Humor`, `Meme`, `Funny` 등)를 통해 관련성 높은 밈을 선별합니다.\n*   **핵심 요약 (Mock)**: 게시물의 제목, 내용, 댓글을 기반으로 밈의 요점을 추출합니다 (현재는 Mock 데이터로 구현).\n*   **슬랙 자동 공유 (Mock)**: 요약된 밈 정보와 원본 링크를 Slack 웹훅을 통해 지정된 채널에 자동으로 게시합니다.\n*   **주기적 업데이트**: 설정된 간격(예: 1시간)마다 자동으로 새로운 밈을 탐색하고 공유합니다.\n\n---\n\n## 🛠️ 시작하기\n\n이 프로젝트는 파이썬 3 환경에서 실행됩니다. 처음 사용하는 초보자도 쉽게 따라 할 수 있도록 상세하게 안내해 드립니다.\n\n### 📝 필수 전제 조건\n\n*   **Python 3.x**: 파이썬이 설치되어 있지 않다면, [Python 공식 웹사이트](https://www.python.org/downloads/)에서 최신 버전을 다운로드하여 설치해 주세요. (설치 시 "Add Python to PATH"를 꼭 체크해 주세요!)\n*   **`pip`**: 파이썬 패키지 관리자입니다. 파이썬 설치 시 함께 설치됩니다.\n\n### ⚙️ 설치 및 설정\n\n1.  **프로젝트 저장소 복제 (Clone)**\n\n    명령 프롬프트(Windows) 또는 터미널(macOS/Linux)을 열고 다음 명령어를 입력하여 프로젝트 파일을 다운로드합니다.\n\n    ```bash\n    git clone https://github.com/your-username/ReddAI-Radar.git\n    cd ReddAI-Radar\n    ```\n\n    (만약 `git`이 설치되어 있지 않다면, [Git 공식 웹사이트](https://git-scm.com/downloads)에서 설치하거나, GitHub 페이지에서 `Code` 버튼을 눌러 ZIP 파일을 다운로드 후 압축을 풀어주세요.)\n\n2.  **가상 환경 설정 및 활성화**\n\n    프로젝트마다 독립적인 파이썬 환경을 구축하는 것은 매우 중요합니다. 가상 환경을 사용하면 시스템 파이썬 환경을 오염시키지 않고 필요한 라이브러리만 설치할 수 있습니다.\n\n    *   **가상 환경 생성:**\n        ```bash\n        python -m venv venv\n        ```\n        (이렇게 하면 `venv`라는 이름의 폴더가 생성되고 그 안에 가상 환경이 설정됩니다.)\n\n    *   **가상 환경 활성화:**\n        *   **Windows:**\n            ```bash\n            .\\venv\\Scripts\\activate\n            ```\n        *   **macOS / Linux:**\n            ```bash\n            source venv/bin/activate\n            ```\n        가상 환경이 활성화되면 터미널 프롬프트 앞에 `(venv)`가 표시됩니다.\n\n3.  **필요한 라이브러리 설치**\n\n    가상 환경이 활성화된 상태에서, 이 프로젝트에 필요한 파이썬 라이브러리들을 설치합니다. (현재 코드는 Mock 함수만 포함하고 있으나, 실제 연동을 위해 `requests` 및 `python-dotenv` 라이브러리가 필요합니다.)\n\n    ```bash\n    pip install requests python-dotenv\n    ```\n    (만약 `requirements.txt` 파일이 있다면 `pip install -r requirements.txt` 명령어를 사용합니다.)\n\n4.  **환경 변수 설정 (`.env` 파일 생성)**\n\n    `SLACK_WEBHOOK_URL`과 같은 민감한 정보는 소스코드에 직접 노출하지 않고 환경 변수로 관리하는 것이 보안상 매우 중요합니다.\n\n    프로젝트 루트 폴더( `ReddAI-Radar` 폴더)에 `.env` 파일을 생성하고 다음 내용을 추가합니다.\n\n    ```\n    SLACK_WEBHOOK_URL="YOUR_SLACK_WEBHOOK_URL_HERE"\n    ```\n\n    *   **Slack Webhook URL 얻는 방법:**\n        1.  Slack 워크스페이스에 로그인합니다.\n        2.  [Slack API 웹사이트](https://api.slack.com/apps)로 이동하여 "Create an App"을 클릭합니다.\n        3.  "From scratch"를 선택하고 앱 이름과 워크스페이스를 지정합니다.\n        4.  좌측 메뉴에서 "Incoming Webhooks"를 선택하고 활성화(On)합니다.\n        5.  "Add New Webhook to Workspace" 버튼을 클릭하고, 봇이 메시지를 보낼 채널을 선택합니다.\n        6.  생성된 "Webhook URL"을 복사하여 `.env` 파일의 `YOUR_SLACK_WEBHOOK_URL_HERE` 부분에 붙여넣으세요.\n\n5.  **설정 값 조정 (선택 사항)**\n\n    `main.py` (또는 프로젝트의 메인 파이썬 파일)를 열어 다음 설정 값들을 필요에 따라 조정할 수 있습니다.\n\n    ```python\n    # REDDIT_SUBREDDITS = ["memes", "dankmemes", "funny", "wholesomememes"] # 밈을 가져올 Reddit 서브레딧 목록\n    # KEYWORDS_TO_FILTER = ["meme", "funny", "joke", "lol", "reaction", "relatable"] # 게시물 제목/내용 필터링 키워드\n    # FLAIRS_TO_FILTER = ["Humor", "Meme", "Funny", "Image", "Video"] # 게시물 플레어 필터링\n    # SCAN_INTERVAL_SECONDS = 60 # 봇이 밈을 스캔하는 주기 (초 단위). 테스트 시 60초, 실제 사용 시 3600초 (1시간) 권장.\n    # MAX_POSTS_PER_RUN = 3 # 한 번 실행 시 슬랙에 보낼 최대 밈 개수\n    ```\n\n    **주의:** `SLACK_WEBHOOK_URL`은 `main.py` 파일 내에서 직접 설정하는 대신, `.env` 파일에서 로드하는 방식으로 구현될 예정입니다. (`python-dotenv` 라이브러리 사용).\n\n### ▶️ 봇 실행하기\n\n모든 설정이 완료되었다면, 가상 환경이 활성화된 상태에서 다음 명령어를 입력하여 셜록봇을 실행합니다.\n\n```bash\npython main.py\n```\n\n봇이 성공적으로 실행되면 터미널에 스캔 주기마다 "Mocking Reddit API call..." 메시지가 출력되는 것을 확인할 수 있습니다.\n\n---\n\n## ⚠️ 경고 및 주의사항 (필독!)\n\n본 프로젝트는 현재 **Mock (가짜) 데이터**를 사용하여 Reddit API 호출 및 Slack 전송을 시뮬레이션하고 있습니다. 실제 Reddit API 및 Slack Webhook 연동은 구현되어 있지 않습니다.\n\n*   **Mock 데이터 사용**: 현재 `get_mock_reddit_posts` 함수는 실제 Reddit에서 데이터를 가져오지 않고, 미리 정의된 샘플 데이터를 반환합니다. 이 프로젝트를 실제 환경에서 사용하려면, [PRAW (Python Reddit API Wrapper)](https://praw.readthedocs.io/en/stable/) 같은 라이브러리를 사용하여 Reddit API와 연동하는 로직을 직접 구현해야 합니다.\n*   **Slack 연동 (Mock)**: Slack으로 메시지를 보내는 부분 또한 현재 Mock 처리되어 있습니다. 실제 Slack 연동을 위해서는 `requests` 라이브러리를 사용하여 Slack Webhook URL로 POST 요청을 보내는 코드를 추가해야 합니다.\n*   **API 한도 및 사용 정책**:\n    *   **Reddit API**: 실제 Reddit API를 사용할 경우, API 호출 빈도에 대한 엄격한 제한이 있습니다. 무분별한 호출은 IP 차단 또는 API 접근 제한으로 이어질 수 있습니다. Reddit의 [API 사용 정책](https://github.com/reddit-archive/reddit/wiki/API)을 반드시 준수해야 합니다.\n    *   **Slack Webhook**: Slack Incoming Webhook에도 초당 요청 수 제한이 있을 수 있습니다. 과도한 메시지 전송은 슬랙 관리자에 의해 제한될 수 있으므로, `SCAN_INTERVAL_SECONDS` 및 `MAX_POSTS_PER_RUN` 값을 신중하게 설정해야 합니다.\n*   **데이터 손실 위험**: 이 프로젝트는 데이터를 영구적으로 저장하지 않습니다. 봇이 재시작되거나 중단되면 이전 스캔 기록이 유지되지 않습니다. (이는 밈 탐지 봇의 특성상 큰 문제가 아닐 수 있습니다.)\n*   **보안**: `SLACK_WEBHOOK_URL`과 같은 민감한 정보는 절대로 소스코드에 직접 노출하지 마세요. `.env` 파일을 통해 환경 변수로 관리하는 것을 강력히 권장합니다.\n*   **AI 요약의 한계**: 현재 코드에는 AI 요약 기능이 구현되어 있지 않습니다. 실제 AI 요약을 적용할 경우, AI 모델의 특성상 항상 정확하거나 의도된 요약이 나오지 않을 수 있습니다. 밈의 맥락을 완벽하게 파악하기 어려울 수 있으므로, 이를 인지하고 활용해야 합니다.\n*   **확장성 및 오류 처리**: 현재 코드는 기본적인 동작을 위한 최소한의 구조를 가지고 있습니다. 실제 프로덕션 환경에서는 견고한 오류 처리, 로깅, 비동기 처리, 그리고 잠재적 예외 상황에 대한 대비가 필요합니다.\n\n---\n\n## 🤝 기여 방법\n\n이 프로젝트는 여러분의 기여를 환영합니다! 개선 사항, 버그 수정, 새로운 기능 제안 등 어떤 형태의 기여라도 좋습니다.\n\n1.  이 저장소를 Fork 합니다.\n2.  새로운 기능 브랜치를 생성합니다 (`git checkout -b feature/AmazingFeature`).\n3.  변경 사항을 커밋합니다 (`git commit -m 'Add some AmazingFeature'`).\n4.  브랜치에 Push 합니다 (`git push origin feature/AmazingFeature`).\n5.

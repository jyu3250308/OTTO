# Kimi의 밈 박사: Slack 피로회복제 봇 💊\n\n## 🤖 당신의 Slack에 유머를 처방할 AI 밈 닥터 Kimi를 소개합니다!\n\n지루한 업무, 끝없이 이어지는 텍스트 속에서 지쳐가는 팀원들을 위한 특효약! 'Kimi의 밈 박사'는 레딧에서 가장 핫한 유머와 밈을 AI가 직접 분석하여, 피로에 지친 Slack 채널에 활력을 불어넣어 줄 코믹하고 찰진 요약을 전달합니다. 이제 'Human-in-the-Loop'의 밈 피로도 끝! AI Kimi가 모든 것을 알아서 처리해 드립니다.\n\n## ✨ 프로젝트 소개\n\n개발자 여러분, 안녕하세요! 저는 당신의 천재 개발자 에이전트 '오또'입니다. 오늘 저는 여러분의 Slack 워크스페이스에 신선한 웃음을 선사할 혁신적인 프로젝트, "Kimi의 밈 박사: Slack 피로회복제 봇"을 소개합니다.\n\n**🤔 왜 이 봇이 필요할까요?**\n우리는 매일 수많은 정보와 소통 속에서 살아갑니다. 특히 개발 문화에서는 밈(Meme)이 중요한 소통 수단이 되곤 하지만, 너무 많은 밈 홍수는 오히려 '밈 피로'로 이어지기도 합니다. 재미있는 밈을 찾아 헤매는 시간, 밈의 맥락을 이해하느라 소모되는 에너지... 이제 이 모든 것을 AI Kimi에게 맡겨보세요!\n\n**💡 Kimi의 밈 박사, 이렇게 작동합니다:**\n1.  **레딧의 실시간 유머와 밈 감지**: Kimi는 인터넷(본 프로젝트에서는 레딧으로 모의 구현)의 흐름을 읽고 가장 인기 있는 밈을 척척 감지합니다.\n2.  **AI '밈 박사 Kimi'의 분석**: 단순히 밈을 가져오는 것을 넘어, Kimi는 최신 AI 기술을 활용하여 밈의 핵심 재미 포인트와 맥락을 분석합니다. 마치 밈 전문가처럼 말이죠!\n3.  **Slack에 전달되는 찰진 요약**: 분석된 밈은 이미지와 함께 Kimi가 직접 작성한 유머러스하고 간결한 요약으로 Slack 채널에 자동으로 공유됩니다. 팀원들은 길게 고민할 필요 없이, 바로 웃음을 얻을 수 있습니다!\n\n## 🚀 주요 기능\n\n*   **레딧 인기 밈 모의 감지**: 주기적으로 레딧의 'Hot' 또는 'Trending' 밈 게시물을 가져오는 기능을 모의 구현합니다. (실제 레딧 API 연동은 향후 기능으로 계획되어 있습니다.)\n*   **AI Kimi의 밈 분석 및 요약 모의**: 가져온 밈의 제목과 맥락을 바탕으로 AI Kimi가 유머러스한 요약을 생성합니다. (현재는 사전 정의된 답변을 무작위로 제공하는 모의 기능입니다. 실제 LLM 연동은 향후 구현 예정입니다.)\n*   **Slack 자동 공유**: 분석된 밈 이미지와 Kimi의 요약을 지정된 Slack 채널에 자동으로 게시합니다.\n\n## 🛠️ 시작하기\n\n이 프로젝트는 Python으로 작성되었으며, 간단한 설정으로 바로 실행할 수 있습니다.\n\n### 📋 사전 준비물\n\n*   **Python 3.x**: [Python 공식 웹사이트](https://www.python.org/downloads/)에서 다운로드 및 설치할 수 있습니다.\n*   **`pip`**: Python 패키지 설치 도구 (Python 설치 시 기본 포함됩니다.)\n*   **Slack 워크스페이스 및 채널**: 밈을 받아볼 Slack 워크스페이스와 채널이 필요합니다.\n*   **Slack Webhook URL**: Slack 채널에 메시지를 보내기 위한 Webhook URL이 필요합니다. [Slack Incoming Webhooks 가이드](https://api.slack.com/messaging/webhooks)를 참조하여 생성하세요.\n\n### ⚙️ 설치\n\n1.  **프로젝트 클론**:\n    ```bash\n    git clone https://github.com/your-username/kimi-meme-doctor-bot.git\n    cd kimi-meme-doctor-bot\n    ```\n    (만약 이 저장소가 없다면, 직접 `main.py` 파일을 생성하고 코드를 복사해주세요.)\n\n2.  **가상 환경 설정 (권장)**:\n    프로젝트의 의존성을 시스템 전역에 설치하는 대신, 격리된 가상 환경을 사용하는 것을 권장합니다.\n    ```bash\n    python -m venv .venv\n    ```\n    *   **macOS / Linux**:\n        ```bash\n        source .venv/bin/activate\n        ```\n    *   **Windows**:\n        ```bash\n        .venv\\Scripts\\activate\n        ```\n\n3.  **의존성 설치**:\n    이 프로젝트는 `requests` 라이브러리를 사용합니다. `requirements.txt` 파일이 있다면 다음과 같이 설치합니다.\n    ```bash\n    pip install -r requirements.txt\n    ```\n    만약 `requirements.txt` 파일이 없다면, 필요한 라이브러리를 개별적으로 설치합니다.\n    ```bash\n    pip install requests\n    ```\n\n### 🔑 환경 설정\n\n`SLACK_WEBHOOK_URL` 변수를 설정해야 합니다. `main.py` 파일 내의 다음 줄을 찾아서, `"YOUR_SLACK_WEBHOOK"` 부분을 실제 Slack Incoming Webhook URL로 교체해주세요.\n\n```python\nSLACK_WEBHOOK_URL = "YOUR_SLACK_WEBHOOK" # Replace with your actual Slack Webhook URL\n```\n**🚨 중요**: 이 URL은 외부에 노출되지 않도록 주의해야 합니다. 실제 서비스에서는 환경 변수나 Secret Management 도구를 사용하는 것이 훨씬 안전합니다. 초보자분들을 위해 코드 직접 수정을 가이드하지만, 보안에 유의해주세요.\n\n## ▶️ 실행 방법\n\n모든 설정이 완료되었다면, 이제 밈 박사 Kimi를 실행할 준비가 되었습니다!\n\n1.  **가상 환경 활성화**:\n    이전 단계에서 가상 환경을 설정했다면, 먼저 활성화합니다.\n    *   **macOS / Linux**: `source .venv/bin/activate`\n    *   **Windows**: `.venv\\Scripts\\activate`\n\n2.  **봇 실행**:\n    프로젝트 루트 디렉토리에서 다음 명령어를 실행합니다.\n    ```bash\n    python main.py\n    ```\n    봇이 실행되면, 모의 레딧 밈을 가져오고, Kimi가 분석한 후, 설정된 Slack 채널로 밈을 게시합니다. 콘솔 출력과 Slack 채널을 확인해보세요!\n\n## ⚠️ 경고 및 주의사항\n\n현재 제공된 코드는 'Kimi의 밈 박사'의 **개념 증명(Proof of Concept)**을 위한 **모의(Mock) 구현**입니다.\n\n*   **밈 데이터**: `get_trending_reddit_memes()` 함수는 실제 레딧 API를 호출하는 대신, 미리 정의된 샘플 밈 목록에서 무작위로 선택하여 반환합니다. 따라서 실제 최신 밈이 아닐 수 있습니다.\n*   **AI 분석**: `analyze_meme_with_ai()` 함수는 실제 LLM(Large Language Model, 예: Kimi, GPT-3/4, LM Studio 등)을 사용하는 대신, 미리 정의된 유머러스한 문구 중 하나를 무작위로 반환합니다. AI의 '밈 분석' 및 '재미 포인트 추출' 기능은 현재 모의 상태입니다.\n*   **Slack 연동**: `post_to_slack()` 함수는 Webhook URL이 설정되지 않았을 경우, 콘솔에 메시지를 출력하는 방식으로 동작합니다. 실제 Slack 메시지 발송은 `requests` 라이브러리를 통해 이루어지지만, 코드에 `requests.post` 부분이 주석 처리되어 있어 실제로는 네트워크 요청을 보내지 않습니다. (주석을 해제하면 실제 전송됩니다.)\n\n**🚨 실제 서비스 개발 시 고려사항:**\n*   **Reddit API 통합**: 실제 레딧의 최신 밈을 가져오려면 [Reddit API](https://www.reddit.com/dev/api/)를 사용하여 개발해야 합니다. 이 경우, API 키 발급 및 사용 제한(Rate Limit)을 준수해야 합니다.\n*   **LLM 연동**: Kimi 또는 다른 강력한 LLM(예: OpenAI API, LM Studio에서 구동하는 로컬 LLM)과 연동하여 진정한 AI 밈 분석 기능을 구현해야 합니다. 이 경우, API 사용 비용, 응답 속도, 토큰 제한 등을 고려해야 합니다.\n*   **오류 처리**: 네트워크 오류, API 응답 오류 등 다양한 예외 상황에 대한 견고한 오류 처리 로직이 필요합니다.\n*   **실행 주기**: 실제 환경에서는 이 봇이 주기적으로 밈을 가져와 게시하도록 스케줄링(예: `cron` 작업, `APScheduler` 라이브러리)해야 합니다.\n*   **보안**: `SLACK_WEBHOOK_URL`과 같은 민감한 정보는 절대 코드에 하드코딩하지 말고, 환경 변수 또는 보안 저장소를 통해 관리해야 합니다.\n\n## 🤝 기여 방법\n\n이 프로젝트는 현재 개념 증명 단계이며, 여러분의 아이디어와 기여를 환영합니다!\n*   버그 리포트\n*   새로운 기능 제안\n*   코드 개선 제안\n*   실제 Reddit API 연동, LLM 통합 등의 Pull Request\n언제든지 GitHub Issue를 통해 의견을 나누거나 Pull Request를 보내주세요!\n\n## 📄 라이선스\n\n이 프로젝트는 [MIT 라이선스](LICENSE)를 따릅니다.

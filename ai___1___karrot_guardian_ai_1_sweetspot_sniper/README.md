# 🥕 AI 당근 파수꾼: 1$ 꿀템 스나이퍼 (Karrot Guardian AI: $1 Sweet-Spot Sniper)

오또의 AI가 당근마켓의 모든 거래를 꿰뚫어봅니다. 단돈 1달러 상당의 꿀템이라도, 설정한 키워드에 딱 맞으면 텔레그램으로 빛보다 빠르게 쏴드립니다! 이제 당신은 '품절' 알림 대신 '득템' 알림을 받으며 1달러의 경제적 자유를 만끽할 수 있습니다!

## ✨ 주요 기능

*   **당근마켓 실시간 스크래핑**: 지정된 키워드를 기반으로 당근마켓 게시물을 주기적으로 스크래핑합니다.
*   **규칙 기반 꿀템 선별**: 설정된 가격(1달러 상당, 기본 1000원) 이하의 저렴한 상품 중, 사용자가 정의한 키워드와 일치하는 '꿀템'을 선별합니다. '무료 나눔' 아이템도 놓치지 않습니다! (본 프로젝트의 'AI'는 복잡한 머신러닝 모델이 아닌, 가격 및 키워드 매칭을 통한 규칙 기반의 '꿀템 선별 로직'을 의미합니다.)
*   **텔레그램 초고속 알림**: 꿀템 발견 시 텔레그램 봇을 통해 즉시 알림을 발송하고, 해당 상품 페이지로 바로 연결되는 링크를 제공합니다.
*   **수익 챌린지 대시보드 (간이)**: 봇이 발견한 총 꿀템 수와 잠재적인 절약 금액을 로그를 통해 실시간으로 보여줍니다.
*   **중복 알림 방지 및 상태 유지**: 한번 알림을 보낸 아이템은 다시 알림을 보내지 않으며, 프로그램 재시작 시에도 이력을 `found_items.json` 파일을 통해 유지합니다.

## 🚀 시작하기

이 프로젝트를 로컬 환경에서 실행하기 위한 단계별 가이드입니다.

### 📋 전제 조건

*   **Python 설치**: Python 3.8 이상 버전이 시스템에 설치되어 있어야 합니다. (파이썬 공식 웹사이트: [python.org](https://www.python.org/downloads/))
*   **pip 확인**: Python 패키지 관리자 `pip`이 제대로 설치되어 있는지 확인합니다. (`python -m pip --version` 명령어로 확인할 수 있습니다.)
*   **텔레그램 봇 토큰**: 텔레그램 봇(`BotFather`를 통해 생성)의 토큰이 필요합니다. ([@BotFather](https://t.me/BotFather))
*   **텔레그램 채팅 ID**: 봇이 알림을 보낼 개인 또는 그룹 채팅의 ID가 필요합니다. 봇과 대화를 시작한 후 [@userinfobot](https://t.me/userinfobot) 또는 [@getidsbot](https://t.me/getidsbot) 같은 봇에게서 쉽게 얻을 수 있습니다.

### 📦 설치 및 설정

1.  **프로젝트 저장소 복제 (Clone)**:
    Git을 사용하여 이 프로젝트 저장소를 로컬 컴퓨터로 복제합니다.
    ```bash
    git clone [프로젝트_저장소_URL_여기에_입력]
    cd karrot-guardian-ai
    ```
    (만약 Git을 사용하지 않는 경우, 프로젝트 파일을 수동으로 다운로드하여 압축을 해제하고 해당 폴더로 이동합니다.)

2.  **가상 환경 생성 및 활성화**:
    의존성 충돌을 방지하고 깔끔한 개발 환경을 위해 가상 환경 사용을 적극 권장합니다.
    ```bash
    # 가상 환경 생성 (폴더 이름은 'venv'로 권장)
    python -m venv venv

    # 가상 환경 활성화
    # Windows 사용자:
    .\\venv\\Scripts\\activate
    # macOS/Linux 사용자:
    source venv/bin/activate
    ```

3.  **의존 라이브러리 설치**:
    `requirements.txt` 파일에 명시된 모든 의존 라이브러리를 설치합니다.
    ```bash
    pip install -r requirements.txt
    ```
    (`requirements.txt` 파일이 없다면, 다음 내용을 복사하여 직접 생성한 후 위 명령어를 실행하세요.)
    ```
    python-telegram-bot==20.7
    beautifulsoup4==4.12.3
    requests==2.31.0
    python-dotenv==1.0.1
    ```
    **참고**: `python-telegram-bot`의 정확한 버전은 `pip freeze`를 통해 확인하거나 공식 문서를 참조하세요. 현재 `20.x` 버전대에서 안정적으로 작동합니다.

4.  **`.env` 파일 설정**:
    프로젝트의 루트 디렉토리(예: `karrot_guardian_ai.py` 파일과 같은 위치)에 `.env` 파일을 생성하고 다음 환경 변수를 설정합니다. 이 파일은 민감한 정보를 안전하게 관리하는 데 사용됩니다.

    *   `TELEGRAM_BOT_TOKEN`: 텔레그램 BotFather ([@BotFather](https://t.me/BotFather))에게서 발급받은 봇 토큰입니다. (예: `1234567890:ABC-DEF1234ghIkl-7890J-1234567890`)
    *   `TELEGRAM_CHAT_ID`: 알림을 받을 텔레그램 채팅 ID입니다. 봇과 대화를 시작한 후 [@userinfobot](https://t.me/userinfobot) 또는 [@getidsbot](https://t.me/getidsbot) 같은 봇에게서 자신의 채팅 ID를 쉽게 얻을 수 있습니다.
    *   `SEARCH_KEYWORDS`: 당근마켓에서 검색할 키워드를 콤마(`,`)로 구분하여 입력합니다. (예: `"아이패드,맥북,의자,무료나눔,급처"`)
    *   `PRICE_THRESHOLD_KRW`: '꿀템'으로 간주할 최대 가격(원)입니다. 이 값 이하의 가격을 가진 아이템만 알림으로 받습니다. (기본값: `1000`)
    *   `SCRAPE_INTERVAL_SECONDS`: 당근마켓을 스크래핑할 주기(초)입니다. 너무 짧게 설정하면 웹사이트에 부담을 주어 IP 차단 위험이 있습니다. (기본값: `300`, 즉 5분)

    **`.env` 파일 예시:**
    ```ini
    TELEGRAM_BOT_TOKEN="YOUR_TELEGRAM_BOT_TOKEN_HERE"
    TELEGRAM_CHAT_ID="YOUR_TELEGRAM_CHAT_ID_HERE"
    SEARCH_KEYWORDS="아이패드,맥북,의자,무료나눔,급처"
    PRICE_THRESHOLD_KRW=1000
    SCRAPE_INTERVAL_SECONDS=180
    ```
    **중요**: 문자열 값(토큰, 채팅 ID, 키워드)은 반드시 큰따옴표(`"`)로 감싸야 합니다. 숫자 값은 큰따옴표 없이 입력해도 됩니다.

### ▶️ 실행 방법

모든 설치와 설정이 완료되었다면, 가상 환경이 활성화된 상태에서 다음 명령어로 봇을 실행할 수 있습니다.

```bash
python karrot_guardian_ai.py
```

봇이 실행되면 콘솔 로그를 통해 현재 상태와 '수익 챌린지' 대시보드를 실시간으로 확인할 수 있습니다. 주기적으로 당근마켓을 스크래핑하고, 설정한 조건에 맞는 꿀템을 발견하면 텔레그램으로 즉시 알림을 보냅니다.

봇을 중지하려면 콘솔 창에서 `Ctrl+C`를 누르세요. 안전하게 종료되며, 다음 실행 시 이전에 발견한 꿀템 정보는 `found_items.json` 파일을 통해 유지됩니다.

## ⚠️ 주의사항 및 한계

*   **스크래핑 정책**: 당근마켓과 같은 웹사이트는 스크래핑을 제한하거나 금지할 수 있습니다. 본 프로젝트는 학습 및 개인적인 사용을 위한 토이 프로젝트이며, 과도한 요청이나 상업적 사용은 웹사이트의 약관을 위반할 수 있습니다. 책임감 있는 사용을 권장합니다.
*   **웹사이트 구조 변화**: 당근마켓 웹사이트의 HTML 구조(클래스 이름 등)가 변경될 경우, 스크래퍼가 정상적으로 작동하지 않을 수 있습니다. 이 경우 `scrape_karrot_market` 함수의 파싱 로직을 업데이트해야 합니다.
*   **지역 필터링**: 현재 `requests` 및 `BeautifulSoup` 기반의 스크래핑은 당근마켓의 동적인 지역 필터링을 완벽하게 구현하기 어렵습니다. 현재 구현은 키워드 기반의 일반 검색에 중점을 둡니다.
*   **속도 제한**: 과도한 요청은 IP 차단으로 이어질 수 있습니다. `SCRAPE_INTERVAL_SECONDS` 값을 최소 300초(5분) 이상으로 설정하여 웹사이트에 부담을 주지 않도록 주의하세요. (코드 내부에서는 최소 60초로 강제하고 있습니다.)
